CRM投資判断レポート

HubSpot導入提案 vs オリジナルCRM継続 — 投資判断と実装ロードマップ
📅 2026-05-14 👤 春川 龍哉 (JapanSauna 事業責任者) 📊 報告先：Sevenrich Group 経営陣
DECISION オリジナルCRM継続を決定。HubSpot提案を不採用とし、4年で約¥774万円のコストを商品開発・広告投資に転用。

📋 エグゼクティブサマリ

  • 背景：月間約150件の問い合わせ、平均成約単価¥150万円。スプレッドシート運用の限界が課題。
  • 選択肢：HubSpot Japan社より月額¥160,000の提案を受領。並行してオリジナルCRMを内製開発中。
  • 結論：オリジナルCRM継続。4年TCO差額：約¥774万円。既に基本機能の8割を実装済み。
  • 独自価値：(1) サウナ販売特化の業務フィット、(2) 後工程（施工〜入金〜粗利）一気通貫、(3) DASH連携で予実管理統合、(4) Claude Opus 4.7 によるAI機能。
  • リスク対応：HubSpot提案の「再検討タイミング検知」機能はMicrosoft Clarity（無料）等で補完。月額¥0〜¥5,000の追加投資で代替実現。

1. 背景と検討経緯

JapanSauna事業（サウナキャビン製造・販売、サウナ施設運営）の事業規模拡大に伴い、現状のGoogleスプレッドシートによる顧客管理体制が限界を迎えている。

現状の業務課題

  • 追客漏れ：月150件の問合せに対し、最終アクションから7日以上経過した「要フォロー案件」が575件中494件（86%）に達する
  • 属人化：担当者の記憶頼みのフォローアップ、ナレッジ共有が困難
  • 再検討検知の不在：失注顧客の再検討タイミングを捕捉できず、機会損失が発生
  • データ分散：マスター/月別/担当者別シートで30以上のシートに分散、二重管理

検討した選択肢

  1. HubSpot導入：Sales Hub + Marketing Hub。HubSpot Japan株式会社（営業担当：石戸 裕大氏）より公式提案受領
  2. オリジナルCRM内製：Next.js + PostgreSQL + Claude API で自社開発（着手済み）

3.2 HubSpot提案を不採用とした個別理由

  • サウナ販売特化機能の不在：製造原価・紹介フィー・施工管理など、HubSpotではカスタマイズが必要で追加開発コストが発生
  • DASH連携不可：Sevenrich Group内部の予実管理基盤との統合がHubSpotでは実現できない
  • AI機能の限定性：HubSpotのBreezeは限定的。Claude Opus 4.7 を自由に組み込める内製のほうが将来性が高い
  • コンタクト数による従量課金：問合せ増加でコストが上振れするリスク
  • データロックイン：HubSpotにデータが集約され、移行困難になる

3.3 HubSpot提案から取り入れる要素

HubSpotの提案資料で最も評価できる点は「再検討タイミング検知」のコンセプト。これは別ツールで以下のように補完する：
