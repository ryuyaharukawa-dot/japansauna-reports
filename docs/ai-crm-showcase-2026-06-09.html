サウナ販売特化CRMを、AIで内製した。

HubSpotをやめ、自分たちの業務に“効く”営業基盤をつくるまで — JapanSauna CRM
📅 2026-06-09 👤 春川 龍哉（JapanSauna 事業責任者） 📊 報告先：Sevenrich Group 経営陣
RESULT HubSpot比 4年で約¥774万円のコストを抑えつつ、サウナ販売特化の営業基盤を内製。問合せメールのAI自動転記まで実現し、本番稼働中。

📋 エグゼクティブサマリ

  • 背景：月約150件の問合せ・平均成約単価¥150万円。スプレッドシート運用が限界で、追客漏れと後工程の分断が課題だった。
  • 選択肢：HubSpot Japan社より月¥160,000の提案を受領。だが「①費用が高い」「②やりたい機能が標準でできない」の2つの壁に直面。
  • 判断：HubSpotを不採用とし、サウナ販売特化のCRMをAI（Claude Code / Claude Opus）で内製。4年TCO差額 約¥774万円。
  • 成果：案件→受注→施工→入金→粗利確定までの一気通貫業務システムを構築。Gmail監視→AIで問合せを自動案件化、計画vs実績の可視化、流入経路別の成約率分析まで実装。
  • 意味：「ツールを導入する」から「業務に合わせて作る」へ。浮いたコストを商品開発・集客に転用できる。

1なぜ作ったのか

JapanSauna事業（サウナキャビン製造・販売、サウナ施設運営）の拡大に伴い、Googleスプレッドシートによる顧客管理が限界を迎えていた。

解決したかった3つの課題

課題 01

追客漏れ・対応遅れ

月150件の問合せに対し、最終アクションから7日以上放置された案件が多発。「誰が・いつ・次に何を」が見えない。

課題 02

情報の分散・属人化

30以上のシートに情報が散在し、最新がどれか分からない。担当者の記憶頼みでナレッジ共有が困難。

課題 03

後工程の分断

受注後（施工・入金・粗利確定）が営業データと切り離され、粗利の着地が見えない。

2HubSpotを検討。だが2つの壁。

解決策としてまずHubSpot導入を検討し、HubSpot Japan株式会社（営業担当：石戸氏）より公式提案（月¥160,000）を受領した。しかし、2つの壁により不採用とした。

壁① 費用が高い（4年TCO比較）

項目HubSpot内製CRM差額
初期費用¥540,000¥0（既存資産活用）¥540,000
初年度ランニング¥1,920,000¥120,000¥1,800,000
2〜4年目¥5,760,000¥360,000¥5,400,000
4年合計¥8,220,000¥480,000¥7,740,000

※HubSpot：月¥160,000×12ヶ月＋初期費用。問合せ（コンタクト）増で従量課金が上振れするリスクあり。
※内製：Vercel ¥3,000/月 ＋ Neon Postgres ¥0〜3,000/月 ＋ Claude API ¥3,000/月 程度（年¥3〜6万）。

壁② やりたい機能が、標準ではできない

HubSpotは汎用CRMとして優秀だが、サウナ販売特有の業務とAIネイティブな作り込みは標準機能の外だった。

やりたかったことHubSpot内製CRM
サウナ販売特化（製造原価・紹介フィー・施工管理）標準になく追加開発が必要初期から組込
後工程の一気通貫（受注→施工→入金→粗利確定）分断・実現困難CRM内で完結
DASH連携（Sevenrich予実管理基盤との統合）実現不可連携可能
AIネイティブ（Claudeで自動転記・失注分析・要約）Breezeで限定的自由に実装
データ所有権HubSpotにロックイン自社保有
HubSpot提案で最も評価できた「再検討タイミング検知」は、Microsoft Clarity（無料）＋メール開封追跡など月¥0〜5,000の補完ツールで代替する方針。コアは内製で持ち、周辺は安価な既製ツールで補う。

3何を作ったか

案件から粗利確定まで、ひとつなぎ

HubSpotでは分断していた“後工程”まで、ひとつのシステムで完結させた。

問合せ→ 通電・商談→ 見積→ 受注→ 施工→ 入金→ 粗利確定

主要機能

🗂️

パイプライン

ドラッグ＆ドロップで状態更新。KPIサマリと加重ヨミを一目で。

🔔

フォローアップ

次アクション期日で管理。対応漏れ・期限切れを自動で可視化。

📊

ダッシュボード

全社/担当者別/月次推移、売上着地見込をリアルタイム表示。

💰

PL × 月次目標

請求書ベースのPLを計画値と連動。計画vs実績・年間ペースを表示。

📈

流入経路分析

meta/Google/SEO/紹介別に件数・成約率・売上を比較。

📝

活動・ドキュメント

架電・商談・見積等の記録と書類を案件単位で蓄積。

技術スタック：Next.js 16 ／ Vercel ／ Neon Postgres ／ Drizzle ORM ／ NextAuth（Google SSO）／ Claude Opus。 開発は Claude Code を活用し、標準的な構成で「誰でも保守できる」状態を維持。

4AIの活用

問合せメールを、AIが案件化する

受信トレイを5分おきに監視し、問合せメールをAIが構造化してCRMに自動登録。手入力ゼロでリードが積み上がる。

Gmail受信→ Fromルールで抽出→ Claude Opusが構造化→ CRMに自動登録→ Slack通知で即共有

そのほかのAI活用

AI 01

AI失注分析

Claude Opus が失注案件のパターンを要約し、改善の示唆を提示。

AI 02

放置案件の自動検知

最終アクションから7日以上の案件を自動でアラート。フォロー漏れを防ぐ。

AI 03

AIで“作る”開発体制

Claude Code活用で、新機能を1日〜数日単位で追加・改善できる。
